experiment.tasks.emg.feedback¶

Perform EMG-feedback

Documentation¶

experiment.tasks.emg.feedback.start(chanOfInterest='EDC_L', numberOfTrials=10, saveDir='C:\\projects\\recording', rwksp='C:\\projects\\config\\bvr_workspace.json')[source]¶

Notes

A visual Emg-feedback task with for a single muscle. It shows a ball representing the current contraction power and a horizontal bar representing the desired target level. The ball turns green when the contraction level is within 10% of the target contraction of 30% MVC.

Parameters:
  • chanOfInterest (the channel used for feedback) –
  • numberOfTrials (A trial consists of 10s feedback and 4s rest) –
  • saveDir (where the runFile will be stored) –
  • rwskp (a json file containing the EEG workspace) –

side effect

saves a Brainvisionrecorder eeg file to saveDir

See also

  • Uses Run to count runs
  • Uses SerialTrigger to send markers
  • Uses Ball for visual feedback

Tip

Requires a json file specific to the target channel and located within the saveDir, which contains the maximum voluntary contraction value. Measure MVC and create such a file with mvc

BVR Markers

  • Ready starts at ‘S 6’
  • Go starts at ‘S 14’
  • Feedback ends at ‘S 10’
