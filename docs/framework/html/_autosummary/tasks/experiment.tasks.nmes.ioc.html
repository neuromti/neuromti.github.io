experiment.tasks.nmes.ioc¶

Measure an Input-Output-Curve

Documentation¶

experiment.tasks.nmes.ioc.start(stimTarget='EDC_L', ISI=(1, 2), repetitions=10, saveDir='C:\\projects\\recording')[source]¶

Notes

Peripheral nerve stimulation with the multiChannelSystems at random stimulation intensities (permutation based) for estimation of input-output-curves.

Parameters:
  • stimTarget (the target muscle used for stimulation) –
  • ISI (a tuple containing minduration & maxduration of interstimulus interval) –
  • repetitions (the number of repetitions of 8) –
  • tacsFreq (the frequency of tACS) –
  • tacsPhase (the phase of tACS at which NMES is applied) –
  • mcsIntensity (the stimulation intensity of NMES) –
  • numProbes (a dictionary with pre, inter and post keys linked to integer values, that define the number of test pulse beforepre and post the run as well as between trials) –
  • numberOfTrials (A trial consists of tACS and paired NMES with test pulses applied inbetween.) –
  • saveDir (where the runFile will be stored) –
  • rwskp (a json file containing the EEG workspace) –

side effect

saves a Brainvisionrecorder eeg file to saveDir

See also

  • Uses Run to count runs
  • Uses multiChannelSystems for stimulation

Tip

Be aware that the stimulators have to be programmed to the desired intensities. There is no programmatic control over these devices besides triggering! Ideally, use the stm files found in ~\ext\mcs\ The multiChannelSystems needs to to be in mf8 mode. Double-check the hardware setup!

BVR Markers

The NMES pulses are applied in mf8 oder. See multiChannelSystems for more details
