experiment.device¶

This module contains classes resembling devices

Overview¶

  • triggerdevices, e.g. i.e. SerialTrigger or SoftwareTrigger
  • screen, i.e. Screen
  • speaker, i.e. Speaker
  • stimulators, e.g. neuroConnDC or multiChannelSystems
  • amplifiers, e.g. BrainAmp or VisionRecorder

class experiment.device.Screen(FontName='inconsolata', selectedMonitor=1)[source]¶

Class representing the display screen for cue presentation

Example:

screen = Screen(FontName = "freesans", selectedMonitor = 1)
screen.open()
# fills monitor 2 with a black background
screen.close()

The screen offers a class-wide dictionary with several default colors (e.g. red, green, gray, etc) at screen.colors. It allows to select fonts and automatically reads the screensize of currently connected displays

class experiment.device.multiChannelSystems(modus='mf8', interface=<experiment.device.ArduinoTrigger object>)[source]¶

Class resembling the multiChannelSystems STG 4000

Notes

Represents the interface to the multichannelsystem STG 4008 from mcs.

A minimal working example:

with xp.device.MCS(modus = 'single') as mcs:
    mcs.trigger(settleDuration = 1)

Triggers the mcs at its current stimulation setting in single mode

Another minimal working example:

with xp.device.MCS(modus='mf8') as mcs:
    for stimKey in mcs.stimDict.keys():
        mcs.trigger(stimKey, settleDuration = 1)
        mcs.sleep(0.05)

Triggers the mcs at each available stimulation settings in mf8 mode

Parameters:modus – can be mf8, mf255 or single

Tip

Beware of possible conflicts between the mcs in mf255 mode and other external devices.

BVR Marker in single mode

  • Trigger: ‘S 1’

BVR Marker in mf8 mode

  • Trigger 1 : ‘S 1’
  • Trigger 2 : ‘S 2’
  • Trigger 3 : ‘S 4’
  • Trigger 4 : ‘S 8’
  • Trigger 5 : ‘S 16’
  • Trigger 6 : ‘S 32’
  • Trigger 7 : ‘S 64’
  • Trigger 8 : ‘S128’

BVR Marker in mf255 mode

  • Trigger 1 : ‘S 1’
  • Trigger 2 : ‘S 2’
  • Trigger 3 : ‘S 3’
  • …
  • Trigger 253 : ‘S253’
  • Trigger 254 : ‘S254’
  • Trigger 255 : ‘S255’

Creates a list of random interstimulus intervals

for stim in range(0,100,1):
    stimKey = mcs.shuffle(count = 1)
    isi = mcs.isi(1, 2, count = 1)
    mcs.trigger(stimKey)
    mcs.sleep(isi)
Parameters:

  • maxduration – maximum duration ISI duration
  • count – number of ISIs to create. Defaults is len of stimDict
Returns:

a list of uniform random ISIs

shuffle(counts: int = None)[source]¶

Permutate keys to stimDict

Returns a list of keys to access the current stimDict in randomly permutated order. It memorizes the already returned keys to maintain permutation across repetitive stimulations.

Example:

for stim in range(0,100,1):
    stimKey = mcs.shuffle(1)
    mcs.trigger(stimKey)
Parameters:counts (int) – Number of permutated keys to generate. Defaults to len of stimDict
Returns:a permutated list of keys for adressing stimDict
Return type:list

See also

reset_shuffle()

class experiment.device.neuroConnDC(stimDuration=None, interface=<experiment.device.SerialTrigger object>, triggerValue=254)[source]¶

Class resembling the neuroConn DC Stimulator

Note

Represents the interface to DC-Plus stimulator from neuroConn.

A minimal working example:

ncdc = neuroConnDC(stimDuration = 15)
ncdc.trigger()

Triggers the neuroconnDC and blocks the script for 15 seconds

Tip

Set the neuroConnDC to listen to its TrigIn. If repetitive blocks of stimulation are planned, add ~7s pause after stimulation to allow the neuroConnDC to return to listening state.

BVR Marker

  • neuroConnDC is marked at ‘S254’ and/or ‘R 1’
